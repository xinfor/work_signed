使用必读

此程序不保证数据的准确性，需仔细进行生成文件的检查！！！

未打卡：深橙色，粗体的“未打卡”
异常打卡：红色，粗体
正常：普通格式

1. 使用方法：
	> 从打卡机获取的Excel内容文件不容修改，改名字为 考勤报表.xls 放入该目录。	注意：文件格式必须为xls后缀，否则会处理失败。
	> 双击执行 生成.bat 文件，即可获得整理的考勤信息为 *月考勤.xls。			注意：得到的考勤并不是最终结果，需据实际进行填写或修改。
	> 为保证准确性，请详细检查生成的报表。

2. 注意事项：
	> 此程序依赖于Python环境，运行需安装Python，包括 xlrd，xlwt两个库。
	> 程序的修改需在专业人员的帮助下进行，不可自己修改。
	> 程序中未进行日期校验，处理中可能有很小的几率出现日期的错位，所以请仔细检查。
	> 发现表格中有重复人员，进行手动删除吧。
    > 由于暂未找到设置字体大小的API，故没有进行修改。
    > 祝君好运！

3. 代码解释：
    > 程序使用 bat 文件进行执行，主程序文件是 attence.py，工具模块是 tools.py。工具模块中包含两个类：AttenceMetaData( 获取考勤的基本信息)，SheetAttenceParse(解析每个Sheet中的考勤)。
    > 由于attence#14直接传入文件名，所以从打卡机中得到的 xls 文件必须重命名为 考勤报表.xls 。可使用读取文件夹中文件进行改进程序。
    > 对于异常或未打卡仅做简单处理，具体的数据依然需要前台实际调查进行填充。

4. bat文件解释：
	> bat 文件可以双击执行，更加傻瓜式操作。bat文件就是cmd命令的集合，双击执行就会一句一句的执行，代码中主要是运行我们的程序: python attence.py
	> @echo off: 关闭提示输出；
	echo：如同print，打印一行；
	color：修改cmd的颜色；
	pause: 暂停；
	cd src: 切换到当前名录的src子目录
	cls: 清屏
	> 由于Windows下默认GB2312编码，所以为执行 生成.bat 不乱码输出，文件编码使用了GB2312，但是使用Spyder以UTF-8编码打开文件，故在Spyder中出现乱码
	> 程序还是很不健壮的，最大的弊端就是未进行异常处理和验证签到的正确性。
	> 改程序可以用于工作之中，但是在确认无误之前，请仔细检查，当几个月考勤的自动生成完全正确之后，方可正常使用。

@echo off
color 0a
echo **********************************************
echo *  请保证从打卡机获取的考勤文件未更改
echo *  并改名字为 考勤报表.xls 该文件夹中
echo *                             make by XiaoY
echo **********************************************
pause
cd src
python attence.py
cls
echo **********************************************
echo *  请保证从打卡机获取的考勤文件未更改
echo *  并改名字为 考勤报表.xls 该文件夹中
echo *                             make by XiaoY
echo **********************************************
echo 完成！
echo 请检查文件考勤文件进行确认是否成功
pause
